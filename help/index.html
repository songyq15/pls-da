Hard and Soft Partial Least Squares Discriminant Analys (PLS-DA) Tool GUI help

Introdiction

The software implementation of hard and soft approaches to Partial Least Squares Discriminant Analysis (PLS-DA) [1] can be used for both multi-class and two-class classification.
The Soft PLS-DA is based on Quadratic Discriminant Analysis applied to the super-score matrix T. It can simultaneously attribute a sample to several classes. It also allows to detect samples, which are not members of any training classes, and, therefore, reduce a number of false positives (e.g. in the presence of outliers).
The Hard PLS-DA represents the conventional PLS-DA classification.

The toolbox provides instruments for data pre-processing as well as for interpretation and visualization of classification models. The main class, PLSDAModel, is responsible for the logic and contains implementation of both methods and auxiliary algorithms. The instance of this class has fields, which represent the actual model, and methods for data visualization and statistics. The PLSDAGUI module provides graphical user interface, which allows user to create and manipulate datasets, calibrate, test and explore models interactively. This class encapsulates all the necessary data such as training or test sets and labels of the samples and variables. The PLS-DA Tool has its own implementation of all necessary statistical functions and does not require the MATLAB Statistics Toolbox.

PLSDAGUI

The PLSDAGUI module provides the graphical user interface for classes DataSet and PLSDAModel. The classes subfolder contains auxiliary classes used by the GUI.

Installation and first steps

To get the latest release plase use GitHub sources. One can clone the git repository or download the source as a zip-file and install it in your Matlab environment.
  • To use the Tool you should set the Matlab current directory to the folder, which contains the Tool's main classes ("PLSDAGUI.m", "PLSDAModel.m", "DataSet.m") as well as all auxiliary files, or add this folder to the Matlab Path.
  • By default it is assumed that none of the built-in Matlab functions on the user's machine has been overloaded due to the use of a third-party toolbox.
  • Before you start working with the GUI you should load the data for analysis into the MATLAB workspace.
  • The data set and model objects can be saved from the GUI to the MATLAB workspace.

Description

The GUI main window is separated into 3 tabs: Data, Model and Prediction. Their availability depends on the user's actions.
On the Data tab user can create and modify the DataSet objects, which can be used for model building, validation and prediction. The functionality provided by the Data tab also includes visual means of Principal Component Analysis.
On the Model tab user can employ one of the previously created DataSet objects to build a Soft or Hard PLS-DA model.
On the Prediction tab user can apply the model to a new or test DataSet.

Examples

Launching the Tool

  1. Load all neccessary data as variables into the current Matlab workspace.
  2. To create a new PLS-DA model, click the New model button.
  3. To work with an existing model load it into the Matlab workspace and click the Existing model button.
  4. The Tool's main window will appear with one or more tabs available.

Working with the Data tab

To create a DataSet object:

  1. Load all neccessary data as variables into the current Matlab workspace.
  2. On the Data tab click the New dataset button.
  3. In the modal window input the Name of the new Data Set and select the Data matrix.
  4. Optionally one can select variables for Classes, Object names, Variable names (or Wavelegths) and Class labels.
    Warning: A DataSet object without Classes can not be used for Calibration!
  5. Click the Add data set button to create the DataSet object.
  6. The newly created object is now available in the Matlab workspace as a variable and in the Data Set drop-down list under the specified name.

To edit or delete a DataSet object:

  1. To edit a DataSet object select it in the Data Set drop-down list and click the Edit button.
  2. In the modal window select the new variables for Data matrix, Classes, Object names, Variable names (or Wavelegths) or Class labels.
    Data fields of the selected DataSet object are shown in the drop-down list as variables with a prefix (equal to DataSet's name).
    Warning: A DataSet object without Classes can not be used for Calibration!
  3. Click the Save button to apply changes to the DataSet object.
  4. The modified object is now available in the Matlab workspace as a variable and in the Data Set drop-down list under the specified name.
  • To delete a DataSet object select it in the Data Set drop-down list and click the Delete button.

To apply preprocessing and analyze a DataSet object:

  1. Load all neccessary data as variables into the current Matlab workspace.
  2. On the Data tab click the New dataset button.
  3. In the modal window input the Name of the new Data Set and select the Data matrix.
  4. Optionally one can select variables for Classes, Object names, Variable names (or Wavelegths) and Class labels.
    Warning: A DataSet object without Classes can not be used for Calibration!
  5. Click the Add data set button to create the DataSet object.
  6. The newly created object is now available in the Matlab workspace as a variable and in the Data Set drop-down list under the specified name.

To manipulate the samples in a DataSet object:

  1. Load all neccessary data as variables into the current Matlab workspace.
  2. On the Data tab click the New dataset button.
  3. In the modal window input the Name of the new Data Set and select the Data matrix.
  4. Optionally one can select variables for Classes, Object names, Variable names (or Wavelegths) and Class labels.
    Warning: A DataSet object without Classes can not be used for Calibration!
  5. Click the Add data set button to create the DataSet object.
  6. The newly created object is now available in the Matlab workspace as a variable and in the Data Set drop-down list under the specified name.

To apply Principal Component Analysis to a DataSet object:

  1. Select the object of interest in the Data Set drop-down list.
  2. In the left part of the Data tab select the PCA sub-tab.
  3. In the modal window input the Name of the new Data Set and select the Data matrix.
  4. Optionally one can select variables for Classes, Object names, Variable names (or Wavelegths) and Class labels.
    Warning: A DataSet object without Classes can not be used for Calibration!
  5. Click the Add data set button to create the DataSet object.
  6. The newly created object is now available in the Matlab workspace as a variable and in the Data Set drop-down list under the specified name.

Working with the Model tab

The Model tab becomes available if at least one DataSet object with Calibration property set to True is present in the Matlab Workspace.

  1. On the Model tab click the Training Set button.
  2. In the modal window select the matrix containing the training set.
  3. Optionally click the Labels button to select the cell array of strings for names of samples in the data set.
  4. Adjust the parameters of the model. It is possible to indicate a value of a type I error or switch to automatic calculation by checking the Auto checkbox
  5. Click the Build button to create the model.
  6. The results are now available in the Current model and Results and statistics sections.
  7. Click on the Acceptance plot and Extreme plot buttons to study the results visually.
  8. To get complete modelling results collected in a DDSimca object click the Save model button and select a name of the new model. It will be created in the Matlab workspace.
  9. You can save your current workspace to a .mat file to preserve the results. Later on you can load the .mat file into the Matlab workspace and resume working with the model by clicking on the Load model button.
  10. Click the Clear button to reset all the setting to default and to remove the current model.

Working with the Prediction tab

The Prediction tab becomes available if the current model is finalized (on the Model tab).

To apply a PLS-DA model to the new data:
Here we assume that a model was previously loaded into the GUI or created on the Model tab.

  1. On the Prediction tab click the New Set button.
  2. In the modal window select the matrix containing the new data set.
  3. Optionally click the Labels button to select the cell array of strings for names of samples in the data set.
  4. By default a type II error is estimated for the new data set. It is possible to switch to a type I error calculation based on the predefined type II error.
  5. Click the Decide button to apply the model to the new data set.
  6. The results are now available in the Results and statistics sections.
  7. Click on the Acceptance plot to study the results visually.
  8. To get the results stored in a DDSTask object click the Save results button and select a name. It will be created in the Matlab workspace. Note that the DDSTask object contains all the data related to the analyzed task including the data sets as well as the related DDSimca model object.
  9. You can save your current workspace to a .mat file to preserve the results. Later on you can load the .mat file into the Matlab workspace and resume the analysis by cliking on the Load results button.
  10. Click the Clear button to reset all the setting to default and to remove the current model.
