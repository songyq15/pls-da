Working with the Cross-Validation tab

The Cross-Validation tab becomes available when the current model is calibrated (on the Model tab).

To apply a PLS-DA model to the new data:
Here we assume that a model was previously loaded into the GUI or created on the Model tab.

  1. On the Prediction tab select a test or new dataset from the New or Test Data Set drop-down list.
  2. Click the Predict button.
  3. One can analyze the results visually on the Graphical view sub-tab.
  4. The Allocation matrix is available on the Table view sub-tab.
  5. In case of a Test dataset (when Classes are known) the Confusion matrix and Figures of merit are also available on the Table view sub-tab.
  6. One can use Save image to file and Copy image to clipboard to store the image in the Graphical view tab to disk or clipboard.
  7. One can use Save tables to file and Copy tables to clipboard buttons to store the Allocation matrix, Confusion matrix and Figures of merit to disk or clipboard.
The Cross-Validation tab contains the settings panel and the right panel two sub-tabs: Data and Results. The settings panel contains the name of the dataset (same as the selected dataset on the Model tab), model parameters and cross-validation specific settings. The dataset, type of the model and the minimum number of PLS components are being shown for your information. The maximum number of PLS components as well as the Type I error can be adjusted on the Model settings panel. The cross-validation is performed for numbers of PLS components from minimum to maximum with the step of one. To perform cross-validation for a range of Type I error you can indicate different values and step on the Model settings panel. You can use controls on the cross-validation settings panel to select the way of splitting the dataset into training and test sub-sets. The following types are available: Leave-One-Out, K-fold, Holdout and Mont-Carlo. Additional input fields will appear when you select different split type. If you click the Split button
